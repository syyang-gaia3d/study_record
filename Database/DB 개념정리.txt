<<개념정리>>

- DB(Database) : 데이터베이스. 정보(데이터)를 저장하고, 컴퓨터를 사용해서 효율적으로 접근할 수 있도록 가공한 것
- DBMS(Database Management System) : 데이터베이스 관리 시스템. 데이터베이스를 관리하는 컴퓨터 시스템

- DBMS가 필요한 이유(특징)
	(1) 다수의 사람이 데이터를 공유하기 쉽다.
	(2) 대량의 데이터를 다루기 쉽다.
	(3) 자동화된 읽기/쓰기 기능을 제공한다.
	(4) 데이터를 안전하게 보존, 관리할 수 있다.
	
- DB의 종류
	(1) 게층형 데이터베이스(Hierarchical Database)
		가장 오래된 데이터베이스 형태. 계층 구조(= 트리 구조)로 데이터를 관리
	(2) 관계형 데이터베이스(Relational Database : RDB)
		가장 폭넓게 이용되고 있는 데이터베이스. 열과 행으로 이뤄진 2차원 표 형식으로 데이터 관리. SQL(Structured Query Language, 구조화 질의 구문)이란 전용 언어를 사용해 데이터 처리(읽기/쓰기 등 지원)
		- RDBMS : 관계형 데이터베이스를 기반으로 하는 DBMS
			-> Oracle(오라클), SQL Server(MS의 RDBMS), DB2(IBM의 RDBMS), PostgreSQL(오픈소스 RDBMS), MySQL(오픈소스 RDBMS)의 5가지가 대표적
	(3) 객체지향 데이터베이스(Object Oriented Database : OODB)
		데이터와 데이터 처리를 모아서 '객체'라는 단위로 관리하는 데이터베이스
	(4) XML 데이터베이스(XML Database : XMLDB)
		XML(인터넷에서 데이터를 교환하기위해 사용하는 형식) 형식 데이터를 대량으로, 빠르게 처리하기 위해 고안된 데이터베이스
	(5) 키-밸류형 데이터스토어(Key-Value Store : KVS)
		검색에 사용하는 키(Key)와 값(Value)을 조합해서 단순한 형태로 데이터를 저장하는 데이터베이스

- 표준 SQL : ISO(국제 표준화 기구)에서 정한 표준 규격을 따르는 SQL <-> 개별 구문 : RDBMS별로 다르게 정의된 SQL

- SQL문과 그 종류
	-> SQL문 : 키워드(처음부터 의미나 사용법이 정해져있는 영단어)와 테이블명, 칼럼명 등을 조합한 하나의 구문. 이를 통해 처리내용을 기술함.
	-> RDBMS에 부여한 명령 종류에 따라 다음의 세 가지로 분류됨
	(1) DDL(Data Definition Language) : 데이터 정의 언어. 데이터를 저장하는 데이터베이스 및 테이블을 생성, 삭제하기 위함.
		- CREATE = 데이터베이스나 테이블 작성
		- DROP = 데이터베이스나 테이블 삭제
		- ALTER = 데이터베이스나 테이블 등의 구성 변경
	(2) DML(Data Manipulation Language) : 데이터 조작 언어. 테이블 행을 검색하거나 변경하기 위함.
		- SELETE = 테이블에서 행 검색
		- INSERT = 테이블에서 신규 행 등록
		- DELETE = 테이블에서 행 삭제
	(3) DCL(Data Control Language) : 데이터베이스에서 처리한 변경 내용을 확정하거나 취소하기 위함. RDBMS 사용자에게 처리 권한 부여.
		- COMMIT = 데이터베이스 변경 내용을 확정
		- ROLLBACK = 데이터베이스 변경 내용을 취소
		- GRANT = 사용자에게 처리 권한 부여
		- REVOKE = 사용자 처리 권한 제거

- SQL 작성 규칙
	(1) SQL문 마지막에는 세미콜론(;)을 붙인다.
	(2) 대소문자 구분이 없으므로 자유롭게 작성할 수 있다. 단, 등록된 데이터는 대소문자를 구분한다.
	(3) 상수 작성법에는 규칙이 있다. 
		- 상수 : SQL 문 안에 직접 기술하는 문자열(문자가 하나 이상 연결된 문자 집합)이나 날짜, 숫자 등
		-- 문자열 : 작은따옴표로 감싼다.
		-- 날짜 : 작은따옴표로 감싸되, 다양한 형식이 있으므로 정해진 형식에 맞춰 내용을 작성한다.(ex : 년-월-일, 년/월/일, 26 Jan 2010 등)
		-- 숫자 : 특별한 기호 없이 숫자만 쓰면 된다.
	(4) 단어는 공백 문자나 줄바꿈 문자로 구분한다. 구분 없이 연결해서 쓰면 에러가 발생한다.

- SQL 명명 규칙
	(1) 데이터베이스나 테이블, 열 등의 이름에 사용할 문자는 [영문자], [숫자], [언더바(_)]만 가능
	(2) -, $, #, ? 같은 기호는 이름으로 사용할 수 없음
	(3) RDBMS 종류에 따라 위의 내용도 사용가능한 경우도 있으나 해당 RDBMS의 독자적 기능으로 다른 RDBMS에선 사용할 수 없을 수가 있음
	(4) 이름의 첫 글자는 '반드시' 영문이어야 함. 숫자나 기호로 시작하는 것은 표준 SQL에서 허락되지 않음
	(5) 이름을 중복해서 사용할 수 없음
		(하나의 데이터베이스 안에 같은 이름의 테이블을 2개 이상 만들거나 하나의 테이블 안에 같은 이름의 칼럼을 2개 이상 만들 수 없음)
